## 13 – Release Notes

### How to Read This Chapter
Release notes track all changes, versions, and compatibility notes. They document what changed and when, but do not reinterpret or restate the framework itself.

### 13.1 VILF 1.0 — Initial Release
Includes:
- full architecture,
- diagnostics,
- operating model,
- governance,
- MEL,
- templates,
- annex structure.

### 13.2 Planned for VILF 1.1
- expanded MEL examples,
- additional archetypes,
- digital repository guidelines,
- domain playbooks,
- capability examples.

Planned items are directional; versioning rules below define how changes are classified.


### 13.3 Versioning Rules
- Major: structural redesign  
- Minor: improvements  
- Patch: corrections  

These rules govern how releases are labeled; contribution guidance follows for external inputs.


### 13.4 Contributions
External suggestions allowed; modifications require permission.
