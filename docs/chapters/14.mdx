## 14 – Roadmap (Future Versions)

### How to Read This Chapter
Outlines planned improvements across short-, medium-, and long-term horizons. It is a forward-looking view, not a release history or execution plan.

### 14.1 Purpose
Provides visibility into future enhancements.

The purpose statement frames intent; the horizon structure below organizes the timeline view.


### 14.2 Horizons
- Horizon 1 (0–12 months): refinements  
- Horizon 2 (12–24 months): archetypes, repositories  
- Horizon 3 (24+ months): cross-country models, advanced MEL  

### 14.3 Features for VILF 1.1
- expanded cases,
- role modules,
- governance audit tools,
- diagrams,
- digital tooling.

These features are directional; later versions extend the roadmap beyond the next release.


### 14.4 Candidate Features for VILF 2.0
- cross-lab data schemas,
- legislation guidance,
- portfolio simulation models,
- multi-country deployment.

Dependencies below indicate the conditions required to validate or prioritize these candidates.


### 14.5 Dependencies
- field testing,
- MEL insights,
- research,
- collaboration.

Governance rules establish how roadmap changes are approved and versioned.


### 14.6 Governance of Roadmap
Must follow VXF versioning rules.
